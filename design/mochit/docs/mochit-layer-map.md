# モチット レイヤーマップ（Rive移行用）

対象ファイル: `design/mochit/source/mochit-layered.svg`
承認済みマスター: `public/characters/mochit/master.png`（1254×1254、アルファ無し・市松模様焼き込み）
実装で使用中の透過画像: `public/characters/mochit/{normal,happy,thinking,cheering}.webp`（512px、透過あり）

## 座標系

- アートボードは **1024×1024・透過**。
- 形状はマスターのピクセル座標系（1254×1254）で作図し、ルートグループ `Mochit_Root` の `scale(0.816587)`（= 1024/1254）で1024アートボードへ収めている。Riveへインポートするとこのスケールはグループ変換として保持される。
- キャラクターの占有域はアートボード上で約 x 160–845 / y 190–845。アンテナ揺れ・バウンス・スクワッシュ＆ストレッチ用の余白を四辺に確保済み。

## 検証方法

マスターから色クラスタ平均・特徴点バウンディングボックスをプログラムで計測し、SVGレンダリング結果とマスターを同座標で重ね合わせて反復調整した。最終的なシルエットは行走査比較で概ね±5px以内、目・口・コア・ハートの特徴点bboxはマスター実測値に一致（1254px基準）。比較資産:

- `design/mochit/references/overlay-compare.html`（ブラウザで開く。不透明度スライダー＋差分モード）
- `design/mochit/exports/mochit-overlay-check.png`（左=マスター / 右=再構成）
- `design/mochit/exports/mochit-layered-preview.png`（1024px透過プレビュー）

## レイヤー構成（下 → 上）

必須グループはすべて独立グループとして存在する。★印は元絵に存在しない**リグ支援用の追加パーツ**（初期状態では不可視 opacity=0、または本体色で視覚的影響なし）。

**v5の構成原則（マスターと同じ「開いた接続」）**: 腕・足・アンテナは本体と**内部線なし**で繋がる。そのため輪郭線は1本の閉曲線ではなく、**分割ストローク**（`Body_Outline` 内の HeadLeft / HeadRight / FlankL / FlankR / BottomDip ＋ 各パーツ群の `*_EdgeOutline` / `*_Edge*`）で描き、接続点でキャップ同士が接線方向に重なることで1本の線に見せている。塗り（`*_Fill`）はすべて本体色の別クローズドシェイプ。

## 再構成の限界（正直な申告）

シルエットはマスターとの行走査比較で概ね±5px以内（1254px基準）まで追い込み済み。残る既知の差分:

1. **アンテナ茎とハートの接続部**: マスターは茎のエッジがハートの輪郭に完全に溶け込む。本ファイルはエッジ線の端をハートの下に埋めて隠しているが、拡大すると接続部に2〜3pxの微小な段差が見える場合がある。Riveでは茎をメッシュ＋ボーンで一本化して解消するのが望ましい。
2. **ハートの形状**: V字ノッチの深い形へ近づけたが、マスターのローブの筒状の膨らみ・ノッチの開き方は手描き特有で、数px単位の差が残る。
3. **足の内側コーナー**: マスターのJフックを再現したが、x520付近の最下点がマスターより数px浅い。
4. **影・グロスのエッジの柔らかさ**: 影は実測形状＋`feGaussianBlur`(3〜4px)。**Riveインポートでフィルタは失われる**ため、Rive側でFeatherを再設定すること（頬はグラデーションなのでそのまま移植可）。
5. **線の強弱**: マスターの輪郭線には手描きの太さの揺らぎがある。本ファイルは均一幅（本体11 / コア外9・内7 / 回路10 / 口13）。
6. **口バリエーション（Smile/Thinking/Open）・舌色・まぶた・白目・Core_Glow**: マスターに存在しない追加パーツ。Neutral表示には一切影響しない（すべて非表示または瞳の下）。
7. **腕・足の裏に隠れる本体側面**: 動かして初めて見える領域は滑らかな補間による推測。

## Riveリグ想定（参考）

- ルートボーン → 体（バウンス・スクワッシュ）→ 頭部（傾き）→ アンテナ3節（遅延追従）/ 目・口（表情）/ 腕L・R（振り）/ 足L・R。
- 瞳・キャッチライトは `attentionX/attentionY` でのポインタ追従を想定し、`EyeWhite` の範囲内で移動。
- `Core_Glow`・`Core_Bar_1..3` は成長段階（`growthStage`）と祝福演出（`triggerCheckpointClear` 等）で制御。
- 命名済みの状態機械コントラクトはアプリ側 `components/mochit/mochitTypes.ts` と `design/mochit/docs/mochit-rive-import-checklist.md` を参照。
