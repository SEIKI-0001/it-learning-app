# ロードマップ地図の画像素材化・UI品質向上

## 目的

`/plan` のロードマップを、既存のチェックポイント進捗を一切変えずに、素材ベースの見やすい冒険地図へ更新する。スマートフォン（390px前後）を最優先に、現在地・到達状況・予定位置を即座に判別できる表示にする。

## 採用する構成

背景地図・霧・ランドマーク・進捗表示を役割ごとに分けるハイブリッド構成を採用する。

- `MapBackground`: `next/image` でベースマップを歪めずに表示する。市松模様が焼き込まれた外周は、既存の `sharp` を使うオフライン処理で透明化した WebP にする。
- `MapRoute`: 既存の Catmull-Rom ベースの道筋をSVGとして保持し、完了区間・未到達区間・新規解放時の道の伸長を描く。
- `MapNodes`: HTMLボタンでステージ番号、状態、地名、現在地、予定位置、達成度を重ねる。タップ領域は44px以上にし、キーボード操作と既存aria-labelを維持する。
- `MapFog`: 白背景を外周から透明化した霧画像を未到達フェーズへ配置する。透明化の見栄えが悪い場合はSVG霧に安全にフォールバックする。解放時の霧晴れ演出とreduced-motion時の静止状態を維持する。
- `MapDetailSheet`: 選択したステージの対応ランドマークを主画像として表示し、正式フェーズ名、説明、チェックポイント、進捗を保持する。
- `mapConfig`: 8ステージとゴールの座標、地名、素材パス、素材サイズを一元管理する。

`components/RoadmapMap.tsx` は外部インターフェースを変えないエントリーポイントとし、選択状態と既読解放演出の制御だけを持つ。

## 素材と表示方針

- ベースマップは `public/maps/roadmap/base-map.webp` に配置する。元画像（1100x1429）は表示領域の既存縦長比率とほぼ一致するため、島を歪ませずに使う。
- 霧は `public/maps/roadmap/effects/fog.webp` に配置する。自然なアルファを確保できた場合のみ地図上で使う。
- 推奨の8ランドマークを `public/maps/roadmap/landmarks/` に配置する。背景地図に同種の地物がすでに描かれているため、地図上には重ねず詳細シートおよび現在ステージの強調に使用する。
- `forest-alt.webp` は使わない。標準の `forest.webp` と二重管理になるため。

## 保持する振る舞い

- `buildCheckpointRoadmap`、`expectedPhaseId`、`done / current / upcoming / goal` の判定は変更しない。
- 「いまここ」「予定ではこのあたり」、ステージ番号、クリア済みチェック、現在フェーズの達成度バーを保持する。
- ノード選択時の詳細シート、霧が晴れる演出、道が伸びる演出、`prefers-reduced-motion`、キーボード操作、aria-labelを保持する。
- 画像のロードに失敗しても、ノードボタンと詳細情報は常に操作可能にする。

## レイアウトとアクセシビリティ

- 地図は最大幅を抑え、デスクトップで引き伸ばさない。
- 390px幅では地名の位置を設定で補正し、説明文を地図上へ追加しない。
- 状態別に色・リング・チェック・透明度を併用し、背景画像上でも識別できるようにする。
- 背景画像には適切な `sizes` と優先読み込みを設定し、詳細画像は必要時に遅延読み込みする。

## テストと検証

- `mapConfig` の8ステージとゴールについて、名称・素材・座標をテストする。
- `done / current / upcoming / goal` の状態表示、`expectedPhaseId`、ノード選択による詳細シート、画像エラー時の操作可能性をコンポーネントテストする。
- reduced-motion時に霧晴れと道の伸長アニメーションが停止することを検証する。
- 関連テスト、全テスト、lint、typecheck、production buildを実行する。
- Playwrightまたはブラウザで390px前後とデスクトップのスクリーンショットを確認し、重なり・ピンずれ・切れ・霧の不自然さを調整する。

## 非対象

学習計画、チェックポイント判定、進捗保存、フェーズ定義、課金、その他の画面は変更しない。
