# モチット統合デザイン

## 目的

人型アバター機能を小型ペットキャラクター「モチット」に完全置換する。モチットは学習を主役に保った控えめな学習相棒として、次の行動、回答結果、チェックポイント達成、成長とバッジを示す。

## 固定する方針

- 人型のプリセット選択、着せ替え、装備、装備解放は廃止する。
- 既存のバッジ、チェックポイント、XP、学習履歴の判定ロジックは変更しない。
- モチットの成長段階はクリア済みチェックポイント数から導出し、過去に到達した段階を取り消さない。
- 正答率・合格率の低下はモチットの成長に影響させない。
- 永続化が必要なモチット専用パラメータは作らない。旧 `checkpointProgress.avatar` は正規化時に除去し、既存の進捗JSONとの互換性を保つ。

## 画像資産

### マスター

`public/characters/mochit/master.png` に受領したPNGを保存する。受領画像は市松模様が焼き込まれておりアルファチャンネルを持たないため、マスターは原本として保管する。

### 実装用の状態画像

`normal`、`happy`、`thinking`、`cheering` を、マスターを編集対象にしたImageGen編集で作成する。顔、体形、輪郭、アンテナ、胸の知識コア、白・紺・水色・ミントの配色を固定する。差分は表情、腕、体の傾きのみとする。

各状態画像は、単色クロマキー背景で生成してローカルで背景を除去し、512px前後の透明WebPとして保存する。

```
public/characters/mochit/
  master.png
  normal.webp
  happy.webp
  thinking.webp
  cheering.webp
```

禁止事項は、3D風表現、厚塗り、過剰な発光、複雑な装飾、文字、床、影、吹き出し、背景エフェクト、別画風である。

## UIコンポーネント

`components/mochit/Mochit.tsx` を唯一の描画窓口とする。

```ts
export type MochitState = "normal" | "happy" | "thinking" | "cheering";
export type MochitSize = "small" | "medium" | "large";
export type MochitAnimation = "idle" | "bounce" | "tilt" | "celebrate" | "none";
```

```tsx
<Mochit
  state="happy"
  size="medium"
  message="いいね。知識がつながってきた！"
  animation="bounce"
/>
```

CSSアニメーションは `transform`、`opacity`、`filter` のみで実装する。`prefers-reduced-motion: reduce` ではすべての動きを停止する。

| 状態 | 演出 | 用途 |
| --- | --- | --- |
| `normal` | ごく弱い上下移動 | 通常表示・学習開始 |
| `happy` | 1回の跳ね | 正解・全問正解 |
| `thinking` | 小さく傾く | 不正解を含む結果、励まし |
| `cheering` | 短い拡大と知識コアの発光 | チェックポイント達成 |

## 画面統合

### 学習ホーム `/today`

「今日やること」の最優先タスクカードに、100〜140pxのモチットを置く。表示はカード内部に限定し、画面右下への固定表示は行わない。セリフはタスクの有無と最優先タスクに応じて、次の行動を一文で示す。

- タスクあり: 「今日はこの3問から始めよう」など
- タスクなし: 「次のチェックポイントまであと少し」など

### 問題回答後

`TopicCompletionQuiz` の完了結果で回答集計とチェックポイントの前後差分を比較し、モチットを結果カード内に表示する。

- チェックポイントが新たにクリア: `cheering`
- 全問正解: `happy`
- 誤答を含む: `thinking` と「惜しい。考え方を一緒に整理しよう」

不正解を悲しませる、または利用者を責める文言は使用しない。

### マイページ `/avatar`

ルートは維持し、タイトルと導線を「モチット」に更新する。200〜280pxのモチット、現在の成長段階、次の成長条件、獲得済みバッジ、解放済みの視覚表現を表示する。外見選択、装備一覧、装備変更UIは表示しない。

### その他の旧アバター表示

チェックポイント最終問題、進捗、ランク、ホームの既存アバターカードをモチットの共通表示へ置換、または役割が重複する場合は削除する。`/more`、下部ナビゲーション、関連ラベルを「モチット」へ更新する。

## 成長と報酬の連携

成長段階は `clearedCheckpointIds` の累積数から決める。成長表現は画像進化ではなく、知識コアの点灯、控えめな光彩、成長段階ラベル、バッジ、チェックポイント達成数を使う。

バッジ獲得とチェックポイント判定は既存の `lib/checkpoints.ts`、`lib/badges.ts`、`lib/studySession.ts` を唯一の真実として再利用する。旧装備解放の通知はバッジ通知へ集約し、同一成果に対する重複通知を作らない。

## 削除対象

- `types/avatar.ts`
- `lib/avatarGrowth.ts`、`lib/avatarItems.ts`、`lib/avatarPresets.ts`、`lib/avatarUnlocks.ts`
- `components/avatar/` 配下の人型アバター・装備・解放通知コンポーネント
- これらを参照するオンボーディング、レイアウト、進捗、ランク、チェックポイント画面の旧アバター連携

削除後は、型、状態マージ、永続化正規化、通知に残る旧アバター参照を取り除く。

## テストと検証

- モチットの状態・画像・アニメーション選択をユニットテストする。
- チェックポイント差分により `cheering` が優先されることをテストする。
- 成長段階がクリア済みチェックポイントの増加に対して単調増加であることをテストする。
- 型チェック、Lint、Vitest、Next.jsビルドを実行する。
- モバイル幅でホーム、問題結果、モチットページを確認し、固定表示がないことを確認する。
